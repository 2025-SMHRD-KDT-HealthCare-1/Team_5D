# Admin 기능 정리 (화면/컴포넌트/데이터)

## 1) 화면/페이지 단위 요약
- 관리자 로그인 화면: 이메일/비밀번호 로그인, 세션 만료 안내.
- 접근 권한 없음 화면: ADMIN 권한 없을 때 안내 및 로그아웃.
- 대시보드: 요약 KPI 카드, 고위험 알림 리스트, 장치 상태 요약.
- 이벤트 모니터링: 실시간 이벤트 테이블/상세, 상태 변경, 알림 전송 토글 등.
- 대상자 관리: 대상자 리스트/상세/수정, 이벤트 히스토리, 보호자 연결 관리, 소프트 삭제.
- 장치 관리: 장치 리스트/등록/상세, 원격 재시작/등록 해제/삭제(소프트), 오류 이력.
- 통계/분석: 기간별 이벤트 차트/비율/추이/오류 유형/대상자 통계.

## 2) 컴포넌트별 기능 요약
- `admin/App.tsx`
  - 관리자 로그인(로컬스토리지 세션), 권한 확인, 화면 전환 상태 관리.
- `admin/components/Layout.tsx`
  - 사이드바 메뉴, 헤더, 상태 표시/재시도 UI.
- `admin/components/Dashboard.tsx`
  - 요약 지표 및 최근 고위험 알림, 장치 상태 요약.
- `admin/components/Monitoring.tsx`
  - 이벤트/대상자/장치 탭 통합 모니터링 UI.
  - 이벤트 상세 관리(상태 변경, 메모, 보호자 알림 토글).
  - 대상자 관리(정보 수정, 보호자 연결, 소프트 삭제).
  - 장치 관리(등록, 상태/오류/이벤트 통계, 소프트 삭제).
- `admin/components/Statistics.tsx`
  - 기간 선택(일/주/월) 기반 차트 및 통계 카드.
- `admin/components/Settings.tsx`
  - 관리자 계정 정보 편집 UI(저장 버튼).

## 3) 데이터/모듈/서비스 요약
- Admin은 컴포넌트 내부 mock 데이터/상태로 동작.
  - Dashboard/Monitoring/Statistics 모두 내부 상수/목업 기반.
- 인증은 로컬스토리지 기반(더미 계정):
  - 이메일: `soin123@naver.com`, 비밀번호: `1234`
  - TTL 30분 만료 로직 포함.

## 실행 관련
- 개발 서버: `npm run dev:admin`
- 빌드: `npm run build:admin`
- 프리뷰: `npm run preview:admin`
